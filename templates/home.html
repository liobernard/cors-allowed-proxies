Need a list of proxies for web scraping or other purposes? sslproxies.org is a great resource, but its web server does not have CORS headers configured to accept cross-origin requests. This shouldn't be a problem server-side, but when attempting a GET request on the client-side via web browser (e.g. a React.js component that fetches page info from sslproxies.org on button press) you'll likely be hit with an error informing you that your browser has blocked the request.

So here's an alternative - a neat proxy server that will make the request for you! Cross-origin GET requests are allowed here, so instead of sslproxies.org, just have your client-side component make requests to the super simple API explained below. The server retrieves the payload from sslproxies.org without issue, then parses to a useful format, and finally gets the proxies back to you.


Usage

/proxies/     Returns an array of <= 100 elite/highly anonymous SSL proxies
/proxies/csv/     Returns a CSV-formatted file of <= 100 elite/highly anonymous SSL proxies
/proxy/     Returns one (1) elite/highly anonymous SSL proxy, tested for workability at httpbin.org/ip (may take a few seconds)
